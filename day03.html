# Подготавливаем датафреймы для обучения и проверки
df = pd.read_excel('rate.xlsx')
money = df.curs
past = 4 * 7
future = 7
start = past
stop = len(money) - future

new_df = []
for i in range (start, stop):
    all_columns = money[(i-past):(i+future)]
    new_df.append(list(all_columns))
    
past_columns = []
for i in range(past):
    past_columns.append(f"past_{i}")
    
future_columns = []
for i in range(future):
    future_columns.append(f"future_{i}")
    
transformed_df = pd.DataFrame(new_df, columns=(past_columns + future_columns))
X = transformed_df[past_columns][:-1]
y = transformed_df[future_columns][:-1]
X_test = transformed_df[past_columns][-1:]
y_test = transformed_df[future_columns][-1:]

# Делаем кроссвалидацию для модели MLPRegressor
# Параметру solver ставим значение lbfgs, т.к. по документации на небольших датафреймах (у нас < 500) работает лучше
# До этого провёл кросвалидацию и с разными значениями этого параметра. Лучшие результаты всегда получал при solver = 'lbfgs'
# Впоследствии убрал этот параметр из кросвалидации и перенёс в конструктор, чтобы уменьшить количество итераций
# Многие параметры не включены в кроссвалидацию, т.к. согласно документации они "работают" когда параметр solver <> 'lbfgs'
mlp = MLPRegressor(random_state=42, solver='lbfgs')
hyper_params = [
    {
        'hidden_layer_sizes': [x for x in itertools.product((100, 200, 300), repeat=1)],
        'activation': ('logistic', 'tanh', 'relu',),
        'alpha': [0.000007, 0.000008, 0.000009, 0.00001, 0.000011, 0.000012, 0.000013],
        'max_iter': range(50, 350, 50),
        
    },
]
gs_mlp = GridSearchCV(mlp, param_grid=hyper_params, scoring='neg_mean_absolute_error', verbose=True, n_jobs=4)
gs_mlp.fit(X, y)

Выводы¶

Библиотека Scikit-Learn имеет множество готовых моделей машинного обучения. Как я понял, для определённых задач одни модели подходят хорошо, другие не очень. На данный момент выбирал методом "тыка". Так же Scikit-Learn предлагает большой выбор интсрументов. В работе я использовал GridSearchCV для проведения кроссвалидации и подбора оптимальных параметров. Полистав документацию увидел и другие инструменты. Например для предобработки данных. Как я понял, некоторые модели требовательны к типу и массштабу данных.

Кроссвалидация. У меня ушло достаточно много времени. Чем больше параметров (и их значений) мы проверяем, тем больше итераций происходит при валидации. Некоторые параметры я устанавливал согласно предложениям в документации. Для сокращения времени на кроссвалидацию, я использовал следующий подход:

  1. Валидировал один-два параметра с большим диапазоном значений;
  2. Добавлял следующий параметр, предварительно оставив лучшие значения параметров из предыдущего шага и пару значений "вокруг"

Пункты в ДЗ:

  1. Пробежался по документации Sklearn. Ознакомился со списком моделей, инструментов. Подробно изучал документацию по моделям и методам, которые использовал в п. 3
  2. Подобрал параметры для MLPRegression. Получил точность по МАЕ 0.58 вместо 1.41
  3. Выбрал модель DecisionTreeRegressor и два ансамблевых метода RandomForestRegressor и BaggingRegressor. Подобрал параметры, отличных от значений по-умолчанию

DecisionTreeRegressor - МАЕ: 1.14 -> 0.94 BaggingRegressor - в моём случае так же использовал дерево решений. МАЕ: 1.09 -> 0.92 RandomForestRegressor - МАЕ: 1.11 -> 0.87

Как видно, ансамблевые методы работают немного лучше
